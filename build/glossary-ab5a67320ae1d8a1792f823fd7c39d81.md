# Glossary

## Canonical Command
A stable top-level command in `devflow` that maps to stack-specific execution.

## Extension
A stack adapter module (`devflow-ext-*`) that resolves canonical commands for a toolchain.

## Fingerprint
A deterministic hash input set used to identify container/runtime build identity.

## Parity
Equivalent behavior and verification logic across local and CI environments.
